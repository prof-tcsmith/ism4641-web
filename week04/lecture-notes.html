Welcome to Lecture 4: Functions, Libraries, and Modules

In today’s session, we will explore three key pillars of Python programming that will empower you to write clearer, more efficient, and reusable code. Our discussion will cover:

  • Functions: The building blocks that let you encapsulate code into manageable, reusable pieces.
  • Libraries: Collections of pre-written code that extend Python’s functionality.
  • Modules: Files containing Python code that can be imported to further organize and reuse your code.

These topics are essential for business analytics, as they allow you to build modular programs that can adapt to various business problems—from calculating financial metrics to processing large datasets.

Functions

What Are Functions?

Functions in Python are self-contained blocks of code designed to perform a specific task. They help us:

  • Organize Code: By breaking a complex problem into smaller, manageable parts.
  • Promote Reusability: Write code once and reuse it in different parts of your program.
  • Enhance Readability: By encapsulating logic in named blocks, your code becomes more self-explanatory.

Defining and Calling Functions

To define a function, use the def keyword, followed by the function name and parentheses that can include parameters. The function body is indented.

Example: A Simple Greeting Function

def greet(name):
    """Greets a person by name."""
    print(f"Hello, {name}! Welcome to Python for Business Analytics.")

# Calling the function:
greet("Alice")
Hello, Alice! Welcome to Python for Business Analytics.

Return Values

Functions can return a value using the return statement, which allows the calling code to capture the result.

Example: Calculating a Discounted Price

def calculate_discounted_price(price, discount_percentage):
    """
    Calculate the final price after applying a discount.
    
    Parameters:
      - price (float): The original price of the item.
      - discount_percentage (float): The discount rate as a percentage.
      
    Returns:
      - float: The final price after discount.
    """
    discount_amount = price * (discount_percentage / 100)
    final_price = price - discount_amount
    return final_price

# Business scenario: 15% discount on a $200 item.
original_price = 200
discount = 15
final_price = calculate_discounted_price(original_price, discount)
print(f"The final price is: ${final_price:.2f}")
The final price is: $170.00

Default and Keyword Arguments

Functions can have default parameter values, making certain arguments optional when the function is called. You can also use keyword arguments to specify parameters by name.

Example: Greeting with a Default Greeting

def greet_customer(name, greeting="Hello"):
    """Greets a customer with a default or custom greeting."""
    print(f"{greeting}, {name}!")

greet_customer("Alice")                      # Uses default greeting
greet_customer("Bob", greeting="Good day")   # Uses a custom greeting
Hello, Alice!
Good day, Bob!

Advanced Function Arguments: *args and **kwargs

Python allows you to write functions that accept a variable number of arguments. This is especially useful in business analytics when the number of inputs may vary (for example, processing an arbitrary number of sales figures).

Using *args

*args allows a function to accept any number of positional arguments. Inside the function, these arguments are accessible as a tuple.

Example: Summing an Arbitrary Number of Sales Figures

def total_sales(*sales):
    """
    Calculate the total sales from an arbitrary number of sales figures.
    
    Parameters:
      - sales: A variable number of numeric sales values.
      
    Returns:
      - float: The sum of all provided sales figures.
    """
    return sum(sales)

# Example usage:
print(total_sales(150.0, 200.5, 320.75))  # Output: 671.25
671.25

Using **kwargs

**kwargs allows a function to accept any number of keyword arguments. These arguments are accessible as a dictionary inside the function.

Example: Analyzing Business Metrics

def report_metrics(**metrics):
    """
    Print a report of various business metrics provided as keyword arguments.
    
    Parameters:
      - metrics: Arbitrary keyword arguments representing metric names and their values.
    """
    for key, value in metrics.items():
        print(f"{key}: {value}")

# Example usage:
report_metrics(Revenue="$500K", Profit="$120K", Growth="15%")
Revenue: $500K
Profit: $120K
Growth: 15%

Combining *args and **kwargs

You can combine both *args and **kwargs in the same function definition to handle both types of arguments simultaneously. The conventional order is: 1. Regular parameters 2. *args 3. Default parameters 4. **kwargs

Example: A Flexible Function for Business Reporting

def business_report(title, *args, report_date="Unknown", **kwargs):
    """
    Generate a flexible business report.
    
    Parameters:
      - title (str): The title of the report.
      - *args: A list of key performance indicators (KPIs) as positional arguments.
      - report_date (str): The date of the report (default is "Unknown").
      - **kwargs: Additional metrics provided as keyword arguments.
    """
    print(f"Report Title: {title}")
    print(f"Report Date: {report_date}")
    print("\nKey Performance Indicators:")
    for kpi in args:
        print(f" - {kpi}")
    print("\nAdditional Metrics:")
    for metric, value in kwargs.items():
        print(f" {metric}: {value}")

# Example usage:
business_report(
    "Monthly Sales Report",
    "Total Sales",
    "New Customers",
    report_date="2025-03-01",
    Revenue="$500K",
    Profit="$120K",
    Growth="15%"
)
Report Title: Monthly Sales Report
Report Date: 2025-03-01

Key Performance Indicators:
 - Total Sales
 - New Customers

Additional Metrics:
 Revenue: $500K
 Profit: $120K
 Growth: 15%

Variable Scope

Understanding the scope of variables helps avoid errors and keeps your code maintainable.

  • Local Variables: Declared inside a function and exist only within that function.
  • Global Variables: Declared outside any function and accessible anywhere in the program.

Example:

global_message = "Welcome to our analytics platform."

def display_message():
    local_message = "Enjoy your learning experience!"
    print(global_message)  # Accessible inside the function.
    print(local_message)   # Accessible only within this function.

display_message()
# print(local_message)  # This would raise an error because local_message is not accessible outside.
Welcome to our analytics platform.
Enjoy your learning experience!

Libraries and Modules

Modules and Libraries: The Building Blocks of Reusability

  • Module: A Python file (.py) containing definitions of functions, classes, or variables.
  • Library: A collection of modules grouped together to provide extended functionalities.

These tools save time by allowing you to use pre-existing code for tasks such as mathematical calculations, date manipulations, and data analysis.

Importing Modules

There are several ways to import modules:

  • Import the entire module:

    import math
print(math.sqrt(16))  # Outputs: 4.0
    4.0

  • Import specific functions or variables:

    from math import ceil, floor
print(ceil(4.2))   # Outputs: 5
print(floor(4.2))  # Outputs: 4
    5
4

  • Import with an alias:

    import numpy as np
array = np.array([1, 2, 3])
print(array)
    [1 2 3]

Standard Libraries for Business Analytics

Python’s standard library includes several modules that are useful for business analytics: - math: Mathematical functions. - datetime: Working with dates and times. - statistics: Calculating statistical measures like mean, median, and variance. - os: Interacting with the operating system.

Example: Using the statistics Module

import statistics

data = [23, 45, 67, 89, 12, 45, 67]
mean_value = statistics.mean(data)
median_value = statistics.median(data)

print(f"Mean: {mean_value}")
print(f"Median: {median_value}")
Mean: 49.714285714285715
Median: 45

Creating and Using Your Own Modules

You can create your own modules to organize your functions and reuse code across different projects.

Creating a Custom Module

  1. Create a File: For example, create a file named financial_tools.py.
# financial_tools.py

def calculate_roi(initial_investment, final_value):
    """
    Calculate the Return on Investment (ROI) given an initial investment and 
    final value.
    
    ROI Formula:
      ROI = ((Final Value - Initial Investment) / Initial Investment) * 100
      
    Returns:
      - float: The ROI as a percentage.
    """
    roi = ((final_value - initial_investment) / initial_investment) * 100
    return roi
  1. Import and Use the Module:
# In your main program or another notebook cell:

#import financial_tools

#roi_result = financial_tools.calculate_roi(10000, 12500)
#print(f"ROI: {roi_result:.2f}%")

Tip: Ensure that the module file (financial_tools.py) exists (that you’ve created) in the same folder as this file, and then uncomment the lines in the code above. If you do not have this module, the code above will not run.

Best Practices and Recap

Writing Clean and Reusable Functions

  • Descriptive Naming: Use clear and descriptive names for functions and parameters.
  • Docstrings: Include docstrings immediately after function definitions to explain purpose, parameters, and return values.
  • Single Responsibility: Each function should perform one specific task.
  • Avoid Side Effects: Minimize modifications to global variables inside functions.

Organizing Code with Modules and Libraries

  • Modularity: Group related functions into modules for improved organization.
  • Reusability: Develop modules that can be imported and reused in different projects.
  • Documentation: Document your code thoroughly to facilitate maintenance and collaboration.
