Example: Filtering Customers for a Promotion

    "C003": {"name": "Charlie", "city": "New York", "spent": 150}
}

Tuples

Introduction to Tuples

  • What are they?
    Tuples are ordered, immutable collections of items.
  • Why use them?
    • Data Integrity: Protect data from accidental modification.
    • Performance: Slightly faster than lists in certain scenarios.
    • Dictionary Keys: Tuples can be used as dictionary keys (lists cannot).

Mutability vs. Immutability

  • Mutable Data Structures:
    • Lists and Dictionaries
    • Contents can be changed after creation (adding, removing, modifying elements).
    # Example with a list (mutable)
my_list = [1, 2, 3]
my_list.append(4)  # Add an element
my_list[0] = 0     # Modify an element
print(my_list)

# Example with a dictionary (mutable)
my_dict = {"a": 1, "b": 2}
my_dict["c"] = 3  # Add a key-value pair
my_dict["a"] = 0  # Modify a value
print(my_dict)
  • Immutable Data Structures:
    • Tuples
    • Contents cannot be changed once created.
    # Example with a tuple (immutable)
my_tuple = (1, 2, 3)
# my_tuple.append(4)  # Raises AttributeError
# my_tuple[0] = 0     # Raises TypeError
print(my_tuple)

Creating Tuples

  • Use parentheses () to create tuples.
  • Elements can be of different data types.
# A tuple representing a product
product = ("Laptop", "P001", 999.99, True)

# A tuple of coordinates
coordinates = (10, 20)

# An empty tuple
empty_tuple = ()

Accessing Tuple Elements

  • Access elements by index (starting at 0), just like lists.
  • You cannot modify elements after creation.
product_name = product[0]    # "Laptop"
product_price = product[2]   # 999.99

print(product_name)
print(product_price)

Immutability (in Detail)

  • Key Feature: You cannot add, remove, or modify tuple elements once created.
  • Why is this useful?
    • Data consistency
    • Less prone to errors in large codebases
    • Hashable (can be used as dictionary keys)
# Attempting to modify a tuple leads to an error
# product[1] = "P002"  # TypeError

Tuple Packing and Unpacking

  • Packing: Creating a tuple from separate values.
  • Unpacking: Assigning tuple elements to individual variables.
# Packing
employee = ("John Doe", 35, "Sales Manager")

# Unpacking
name, age, position = employee

print(name)
print(age)
print(position)

Common Tuple Use Cases

  • Data records that must remain unchanged
  • Coordinates or dimension data
  • Function returns (multiple values at once) — (will discuss next week)
  • As dictionary keys for fast lookups

Tuple Methods

  • count(x): Returns how many times x appears.
  • index(x): Returns the index of the first occurrence of x.
numbers = (1, 2, 3, 2, 4, 2)
count_of_twos = numbers.count(2)   # 3
index_of_three = numbers.index(3)  # 2

print(count_of_twos)
print(index_of_three)

Sets

What are Sets?

  • Unordered collections of unique elements.
  • No duplicate values.
  • Efficient membership checks (in operation is very fast).
  • Support mathematical set operations like union, intersection, difference.

Creating Sets

  • Use curly braces {} or the set() function.
# Using curly braces
fruits = {"apple", "banana", "orange", "apple"}  
# Duplicates removed automatically

print(fruits)

# Creating a set from a list
colors = set(["red", "green", "blue", "red"])
print(colors)

Adding and Removing Elements

  • add(x): Adds an element x.
  • remove(x): Removes x, raises an error if x doesn’t exist.
  • discard(x): Removes x if present (no error otherwise).
  • pop(): Removes and returns an arbitrary element.
  • clear(): Empties the set.
fruits.add("grape")
fruits.remove("banana")
fruits.discard("kiwi")  # No error even if "kiwi" is not present

print(fruits)

Set Operations

  • Union (|): Elements in either set.
  • Intersection (&): Elements in both sets.
  • Difference (-): Elements in the first set but not the second.
  • Symmetric Difference (^): Elements in either set but not both.
set1 = {1, 2, 3}
set2 = {3, 4, 5}

union_set = set1 | set2         # {1, 2, 3, 4, 5}
intersection_set = set1 & set2  # {3}
difference_set = set1 - set2    # {1, 2}
sym_diff_set = set1 ^ set2      # {1, 2, 4, 5}

print("Union:", union_set)
print("Intersection:", intersection_set)
print("Difference:", difference_set)
print("Sym. Diff.:", sym_diff_set)

Common Set Use Cases

  • Removing duplicates (e.g., from a list).
  • Checking membership quickly.
  • Finding common elements between two lists or sets.
  • Simplifying “unique item” analyses.

Conclusion

You’ve learned two important data structures that expand your ability to manage data in Python:

  • Tuples: Ordered, immutable sequences that help preserve data integrity and can serve as dictionary keys.
  • Sets: Unordered collections of unique items that excel at membership tests and mathematical set operations.

These tools allow you to organize data in ways that best suit different analytical tasks. They will become even more powerful once you start combining them with more advanced features in Python.
