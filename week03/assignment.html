Introduction

The Summit Exports Scenario

Summit Exports is a fictional logistics company shipping specialized items to various destinations. Your tasks:

  1. Use Tuples to store information about containers (container IDs, coordinates, fragile status, etc.).
  2. Use Sets to manage the list of unique destinations.
  3. Apply loops (such as for or while) and conditional statements (if, elif, else) to demonstrate membership checks and data manipulations in sets.
  4. Present the results in a clear, step-by-step manner, using print statements and f-strings.

You will write your code in a Google Colab notebook, with Markdown cells explaining each step.

Task 1: Container Data with Tuples

  1. Create Three Container Tuples

    • Each tuple might look like:

      container_a = ("CONT-101", 40.7128, -74.0060, True)

      Where:

      • CONT-101 is the container ID (a string).
      • 40.7128 and -74.0060 are the lat/long coordinates (floats).
      • True indicates whether the contents are fragile (Boolean).
  2. Print Each Tuple

    • Show that all three tuples were created properly.

    • For instance:

      print("Container A:", container_a)
print("Container B:", container_b)
print("Container C:", container_c)
  3. Tuple Immutability Check

    • Attempt to modify one of the tuples, for example:

      # container_a[1] = 41.0

    • Catch or show the error that occurs in a try-except block, or comment out the line, and explain in Markdown why it fails.

  4. Unpack a Tuple
    • Pick one of the container tuples and unpack it into separate variables (e.g. cont_id, lat, lng, fragile = container_a).
    • Print each variable individually using f-strings to confirm you’ve successfully extracted the data.

Goal: Demonstrate you understand how to define tuples, that they’re immutable, and that you can unpack them into separate variables for clarity.

Task 2: Managing Destinations with Sets

  1. Create a Set of Destinations

    • For example:

      destinations = {"New York", "Los Angeles", "Dallas", "Chicago", "Miami"}

    • Print the set. (Remember, sets may appear in any order.)

  2. Modify and Check Membership
    • Add a new city to the set (e.g., "Seattle") using .add().
    • Remove a city (e.g., "Dallas") using .remove(). Print the set after each operation.
    • Check membership for a city that might not be in the set (e.g., "Boston") using "Boston" in destinations, and print a statement like Boston is in the set: False/True.
  3. Remove vs. Discard

    • Demonstrate the difference between .remove() and .discard() by calling them on a city that does not exist. For example:

      # Attempt remove()
destinations.remove("Paris")  # Should raise an error if not in the set

# Attempt discard()
destinations.discard("Paris")  # Should not raise an error

    • Print appropriate messages indicating the outcome.

  4. Basic Set Operations

    • Create two additional sets, for instance:

      west_coast = {"Los Angeles", "Seattle"}
east_coast = {"New York", "Miami"}

    • Print the results of:

      • Union of west_coast and east_coast
      • Intersection of destinations and east_coast
      • Difference between destinations and west_coast
      • Symmetric difference between west_coast and east_coast

Goal: Confirm your ability to manipulate sets, remove duplicates, check membership, and perform set operations.

Task 3: Bringing It All Together

Summit Exports needs a combined view of these container tuples and the shipping destinations.

  1. Display Container Details

    • Print each container’s info in a user-friendly format. For example:

      Container A:
  ID: CONT-101
  Coordinates: (40.7128, -74.0060)
  Fragile: True

    • Use f-strings for clarity.

  2. Count Unique Destinations

    • Print how many unique destinations are in your destinations set using len(destinations).

    • To see them in alphabetical order, convert to a list and sort:

      sorted_destinations = sorted(destinations)
print("All destinations (sorted):", sorted_destinations)
  3. Optional Looping / Condition

    • (Optional) Add a small loop that iterates over each destination in destinations. Print a statement like:

      for city in destinations:
    if city in east_coast:
        print(f"{city} is on the East Coast.")
    elif city in west_coast:
        print(f"{city} is on the West Coast.")
    else:
        print(f"{city} is neither East nor West Coast, or is unclassified.")

Goal: Show that you can combine everything into one scenario, referencing your container tuples and sets of destinations to provide a final summary.
