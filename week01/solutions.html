3. Variable Naming

  • my_variable: Valid. Starts with a letter, followed by an underscore and a letter.
  • 1st_place: Invalid. Cannot start with a digit.
  • _score: Valid. Can start with an underscore.
  • totalSum: Valid. Starts with a letter and uses camel case.
  • sales-report: Invalid. Contains a hyphen, which is not allowed in variable names.
  • True: Invalid. True is a reserved keyword in Python (a Boolean value).

5. What’s the Output?

x = 10
y = "5"
print(x + int(y))

Output: 15

Explanation:

  • x is an integer (10).
  • y is a string (“5”).
  • int(y) converts the string “5” to the integer 5.
  • x + int(y) then adds the integer 10 and the integer 5, resulting in 15.

5. What’s the Output?

a = 10
b = 5
c = 2

result = a > b and b < c or a / b == c
print(result)

Output: True

Explanation:

  • Operator precedence: > and < are evaluated before and and or. / and == are evaluated before and and or.
  • a > b evaluates to True (10 > 5).
  • b < c evaluates to False (5 < 2).
  • a / b evaluates to 2.0 (10 / 5).
  • a / b == c evaluates to True (2.0 == 2).
  • The expression becomes: True and False or True
  • and is evaluated before or: True and False is False
  • The expression becomes: False or True which is True

10. More on What’s the Output?

x = 12
y = 7
z = 4

result = x - y > 5 and y / z < 2 or x % z == 0 
print(result)

Output: True

Explanation:

  • Operator precedence: >, <, -, /, %, == are evaluated before and and or.
  • x - y evaluates to 5 (12 - 7)
  • x - y > 5 evaluates to False (5 > 5)
  • y / z evaluates to 1.75 (7 / 4)
  • y / z < 2 evaluates to True (1.75 < 2)
  • x % z evaluates to 0 (12 % 4)
  • x % z == 0 evaluates to True (0 == 0)
  • The expression becomes: False and True or True
  • and is evaluated before or: False and True is False
  • The expression becomes False or True
